# 千机聆阙

作者：雪碧bir

千机聆阙是一个 AstrBot 群聊读空气插件。它旁路监听群消息，先用本地规则和轻量打分判断是否该回复，灰区消息再走短超时 TimingGate 判断节奏，最终沿用 AstrBot 当前会话模型和人格生成回复。

## 功能

- 群聊上下文短期记忆
- 本地快速门控，减少无意义模型调用
- 灰区短超时 TimingGate，降低灰区长期 `wait` 的概率
- 优先使用 AstrBot 标准 `request_llm` 回复管线，保留当前人格与会话
- 已发送/已产出回复后的冷却和连续发言等待
- 中文管理指令

## 指令

- `/读空气 状态`：查看当前群状态
- `/读空气 开启`：启用当前群
- `/读空气 关闭`：关闭当前群
- `/读空气 模式 安静`：更克制
- `/读空气 模式 普通`：平衡模式
- `/读空气 模式 积极`：更积极
- `/读空气 原因`：查看最近一次判定原因

也可以使用别名 `/空气`。
常用短写：`/空气 查`、`/空气 开`、`/空气 关`、`/空气 模 普通`、`/空气 因`。

## 配置

插件通过 AstrBot WebUI 读取 `_conf_schema.json`。默认只在群聊生效，但新安装时不会自动监听任何群；请在目标群使用 `/读空气 开启` 自动写入当前平台实例 key，或在 `enabled_groups` 中显式填写 `*` 才启用所有群。手动填写时推荐使用 `平台实例ID:GroupMessage:群号`；裸群号只适用于没有平台实例信息的旧事件。

`wait` 表示本轮先不回复：连续发言、TimingGate 高置信判断应等待、同群已有回复生成中都会走这个降级。灰区节奏判断默认开启，只对强求助、常见疑问或接近回复阈值的灰区调用一次轻量 LLM，默认 1.5 秒超时；如果节奏判断返回低置信 `wait/ignore`、超时、异常或非法 JSON，但当前灰区消息本身带有强求助信号且分数达到 `llm_gate_fallback_score`，插件会用本地兜底策略接话，默认覆盖约 `0.42` 的求助灰区；弱疑问只会进入 TimingGate，不会在节奏判断关闭时直接兜底；如果 TimingGate 高置信判断应等待或忽略，则尊重它，避免抢话。插件主动请求回复时会抑制 AstrBot 默认 LLM 链路，避免同一条消息被默认模型再答一次；只有消息链 @、引用 bot 或 AstrBot wake 前缀这类明确接管场景会停止事件传播。纯昵称命中不会截断后续插件；如果同一事件里已有其他插件先发起 LLM，本插件会避让不再重复回答；如果同群已有回复生成中，纯昵称追问会安静跳过，不额外发送忙碌提示。可在 WebUI 关闭“接管明确点名”。

短期上下文有容量和过期清理：默认最多保留 200 个群状态，群 1 天无活动后清理。正在生成回复的群会被临时保护到发送确认或超时清理；当同时 pending 的群超过容量上限时，优先保留最近的 pending 群，容量上限仍生效。多 aiocqhttp 实例优先使用 AstrBot 的 `unified_msg_origin` 作为状态 key，同时兼容旧的 `平台实例ID:群号`；裸群号只用于没有平台实例信息的旧事件，避免多实例串群启用。

当前版本优先使用 AstrBot 标准 `ProviderRequest` 会话管线，并透传当前消息链里的基础图片和语音。媒体只接管 AstrBot `Plain`、`At`、`Image`、`Record` 和只读 `Reply` 组件；图片/语音引用仅接受 `base64://`、AstrBot 临时目录内已存在的本地路径，或指向临时目录内已存在文件的 `file:///`。为避免 AstrBot ProviderRequest 组装时产生 MIME 错配，本插件主动接管的图片限制为 JPEG，语音限制为 WAV；`base64://` 编码长度限制为 700000 字符，本地媒体大小限制为 1.5MB。无法识别或格式不匹配的媒体会放行默认链路。复杂引用附件、远程媒体 URL、视频/文件解析、未知组件会放行默认链路；Dify/Coze/DashScope/DeerFlow 等第三方 runner 不走插件主动 ProviderRequest，灰区求助会用灰区短超时尝试短文本直发兜底，失败后放行默认链路，但未明确点名时默认链路未必会触发回复。放行场景本身不会追加用户消息到本插件上下文，但默认链路或其他插件随后真实发出的回复会作为 bot 节奏信号进入短期上下文和冷却，避免刚回复完又插话。

当前发布元数据仅声明已验证的 `aiocqhttp`；其他平台不会被被动监听，开启/关闭/模式等变更类命令也不会写入配置，只读状态类输出可能仍会响应。

默认会输出“读空气判定日志”。AstrBot 日志会记录本插件对群消息的动作、分数、原因，以及 TimingGate、ProviderRequest 正式回复和第三方 runner 直发兜底各自是否实际调用 LLM。默认不会记录群聊原文，只显示消息长度。如需查看最多 80 字消息摘要，可另行打开“日志包含消息摘要”，但这可能暴露隐私内容，建议只在短时间排查时使用；如果群太活跃，也可以关闭“输出读空气判定日志”。

## 安装

```powershell
cd <AstrBot目录>\data\plugins
git clone https://github.com/diaomin66/astrbot_plugin_qianji_lingque astrbot_plugin_qianji_lingque
```

重启 AstrBot 或在 WebUI 重载插件后，在目标群发送 `/读空气 状态` 做烟测，再按需使用 `/读空气 开启`、`/读空气 模式 普通` 调整。

`/读空气 开启`、`/读空气 关闭`、`/读空气 模式` 使用 AstrBot 管理员权限，不等同于 QQ 群管理员权限。

## 开发验证

```powershell
python -m compileall -q main.py qianji_lingque tests
python -m unittest discover -s tests
```
